# Hand Gesture Calculator

## 概要
Webカメラ映像から左右の手の指本数を認識し，ジェスチャー入力で四則演算を行うアプリです．

MediaPipeで手の21点ランドマークを取得し，手首を基準に正規化した63次元特徴量をMLPに入力することで，0〜5本の指本数を6クラス分類しています．

## 背景と目的
画像を入力として扱う仕組みに興味を持ちました。その応用として、個人でMediaPipeとPyTorchを用い．教育支援を想定した手指ジェスチャー認識による四則演算システムが作れるのではないかと考えました．

## 主な機能
・Webカメラによるリアルタイム手検出  
・MediaPipeによる21点ランドマーク取得  
・ランドマーク座標の正規化  
・MLPによる0〜5本の指本数分類  
・左右の手を数値入力として利用  
・`+`, `-`, `*`, `/` の四則演算  
・0除算時のエラー表示  
・直近フレームの多数決による表示の安定化

## 実行方法

python -m venv .venv  
.\.venv\Scripts\activate  
pip install -r requirements.txt  
python run.py  


## 操作方法(キーボードで入力)

・`+` : 足し算  
・`-` : 引き算  
・`*` : 掛け算  
・`/` : 割り算  
・`q` : 終了  
・(`c` ：クイズモード．backupフォルダにあります．)

## データ収集

python collect_finger_landmarks.py  

0〜5のキーを押すことで，現在映っている手のランドマーク特徴量と正解ラベルをCSVに保存します．  

保存先は以下です．  

data/finger_landmarks.csv  

## 学習

python train_finger_mlp.py  

`data/finger_landmarks.csv` を読み込み，MLPを学習します．  

保存先は以下です．  

models/finger_mlp_model.pth  

## 開発過程

`back_up` フォルダには，開発途中のコードや試作機能を保存しています．  

最初はMediaPipeのランドマークを用いた距離計算で指本数を判定していましたが，手の角度や親指の向きによって誤判定が起きやすかったため，ランドマークを正規化した特徴量を用いるMLP分類モデルに改善しました．  

~また，発展機能として計算問題を表示し，ジェスチャーで答えるクイズモードも試作しました．ただし，いざ実装しようとなった場合にリアルタイム映像では同じジェスチャーが複数フレームにわたって検出されるため，正解判定が連続して実行される問題がありました．~  
クイズモードは試作段階として `back_up` に分けています．

・UIの改善  

・LightGBMなどを用いたアンサンブル  
ランドマーク特徴量は表形式データであり，MLPに加えLightGBMなどの表データに強い手法を組み合わせることで，精度向上が期待できるため

・指ごとのON/OFF判定
5本指の組み合わせ(32通り)をクラス分類するより，各指が曲げているか，伸ばしているかの状態を二値分類を行い，5つのバイナリ分類器にする方がデータ効率が良いと考えているため
